thresh.

Point it at a Jira scope and it surfaces the pending, actionable issues, ranked by impact, as a markdown report on stdout. The deterministic core is stdlib-only; an optional LLM tie-break recovers the signal Jira's fields don't encode.

Three ways to run it

Same discovery and ranking engine, three front ends. Pick the one that fits where you are.

--stdout

CLI

Markdown report on stdout. Pipe it to glow, redirect to a file, or paste it into a standup.

thresh ABC-123
--tui

TUI

Navigable list (Bubble Tea). Open in browser, self-assign, transition, or comment, right from the keyboard.

thresh --tui ABC-123
--mcp

MCP server

Exposes discovery and per-issue actions as tools Claude Code can call. The calling model does the re-rank.

thresh --mcp

How the ranking works

Deterministic by default (free, no model). Optional --llm tie-break re-ranks the top N by semantic impact, then falls back to the deterministic order on any error.

▸

Momentum, decayed

In Progress > Selected > Backlog, aged down over time. Subtasks inherit the parent's momentum.

▸

Foundational = structural

An issue ranks up only if it actually blocks others, not because the title sounds infra-y.

▸

Bugs by blast radius

data-loss > broken-flow > degraded > cosmetic, judged from the description, not the title.

▸

Zombie detection

An "In Progress" item untouched for 60+ days is abandoned, and ranked below real live work.

▸

Gated work, flagged

Anything with an open blockedBy is marked and pushed below comparable unblocked work.

▸

Two-pass LLM

Ranks on titles + links first, pulls only the ambiguous descriptions on demand, then re-ranks. Cheap path stays cheap.

Install

Prebuilt binaries for linux / macOS / windows × amd64 / arm64 ship with every v* tag via GoReleaser. Or build from source with Go 1.25+.

Get the binary

# with Go:
go install github.com/alejandroSuch/thresh@latest

# or download for your platform:
github.com/alejandroSuch/thresh/releases

Point it at Jira

# create a token at id.atlassian.com
export THRESH_JIRA_BASE_URL=https://your-org.atlassian.net
export THRESH_JIRA_EMAIL=you@example.com
export THRESH_JIRA_TOKEN=<api-token>

# or store it once:
thresh setup

Wire it into Claude Code (MCP)

claude mcp add thresh -- /abs/path/to/thresh --mcp
# read-only, calling model re-ranks:
claude mcp add thresh -- /abs/path/to/thresh --mcp --no-llm --tools discover_backlog,list_transitions
